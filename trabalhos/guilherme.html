AWS Lambda

Serverless Computing

O AWS Lambda é um serviço serverless onde o desenvolvedor envia apenas funções de código que são executadas automaticamente quando ocorre um evento. Ele funciona sob demanda, criando a infraestrutura apenas no momento da execução e encerrando logo depois, o que reduz custos e simplifica o gerenciamento. O Lambda integra-se com diversos serviços da AWS e permite arquiteturas orientadas a eventos, disparando processos adicionais sempre que necessário. Exemplo prático: um upload de imagem em um bucket S3 pode acionar uma função Lambda que gera versões menores da imagem para uso no site.

Amazon EC2

Elastic Compute Cloud

O Amazon EC2 é um serviço de infraestrutura como serviço que permite criar servidores virtuais na nuvem, chamados instâncias. Essas instâncias podem ser configuradas com diferentes níveis de memória, processamento e armazenamento, atendendo desde testes simples até aplicações empresariais robustas. O EC2 oferece flexibilidade para escolher sistema operacional e softwares, além de elasticidade para aumentar ou reduzir capacidade conforme a demanda. Exemplo prático: uma empresa pode hospedar seu sistema web em uma instância EC2 rodando Linux com servidor e banco de dados instalados manualmente.

Servidores virtuais Infraestrutura

Amazon S3

Simple Storage Service

O Amazon S3 é um serviço de armazenamento de objetos que organiza arquivos em buckets e oferece alta durabilidade e disponibilidade. Cada objeto possui identificador único e metadados, permitindo acesso direto pela internet e facilitando o gerenciamento de grandes volumes de dados. A AWS replica automaticamente os arquivos em diferentes locais físicos, reduzindo a chance de perda de informação. Exemplo prático: sites podem armazenar imagens e vídeos enviados por usuários no S3, enquanto o servidor principal apenas gerencia a aplicação.

Armazenamento Objetos

Amazon DynamoDB

NoSQL Database

O Amazon DynamoDB é um banco de dados NoSQL totalmente gerenciado que utiliza estruturas flexíveis de chave-valor. Ele oferece respostas rápidas mesmo com grandes volumes de dados e ajusta automaticamente sua infraestrutura para manter o desempenho. Além disso, garante alta disponibilidade replicando informações em múltiplas zonas de disponibilidade. Exemplo prático: aplicativos mobile com milhões de usuários podem usar o DynamoDB para armazenar perfis e preferências.

NoSQL Alta performance

Amazon Aurora

Relational Database Service

O Amazon Aurora é um banco de dados relacional compatível com MySQL e PostgreSQL, projetado para unir a confiabilidade da nuvem com a estrutura tradicional SQL. Sua arquitetura distribui dados em vários nós de armazenamento, aumentando a confiabilidade e desempenho das consultas. O serviço também oferece backup automático, recuperação de falhas e replicação sem necessidade de administração manual. Exemplo prático: sistemas de e-commerce podem usar Aurora para armazenar pedidos, clientes e pagamentos com segurança.

SQL Alta confiabilidade

AWS WAF

Web Application Firewall

O AWS WAF é um firewall de aplicações web que protege contra acessos maliciosos e ataques como SQL Injection e Cross-Site Scripting. Ele analisa cada requisição HTTP/HTTPS e aplica regras configuradas para permitir, bloquear ou monitorar padrões de tráfego. Essa camada de proteção é essencial para aplicações expostas na internet, evitando que vulnerabilidades sejam exploradas. Exemplo prático: um portal corporativo pode usar o WAF para bloquear tentativas de acesso suspeitas antes de chegarem ao servidor.

Segurança Firewall

Amazon CloudWatch

Monitoring Service

O Amazon CloudWatch é o serviço de monitoramento da AWS que coleta métricas como uso de CPU, memória e requisições. Ele permite visualizar gráficos em tempo real e configurar alertas automáticos para detectar problemas antes que afetem os usuários. Além disso, armazena e analisa logs de aplicações, facilitando a identificação de erros e comportamentos inesperados. Exemplo prático: se uma instância EC2 ultrapassar o limite de CPU, o CloudWatch envia um alerta imediato aos administradores.

Monitoramento Logs

AWS CodeCommit

Git Repository Service

O AWS CodeCommit é um serviço de hospedagem de repositórios Git totalmente integrado ao ecossistema da AWS. Ele permite que equipes armazenem e acompanhem mudanças no código, mantendo histórico de versões e commits. O CodeCommit facilita pipelines de integração e entrega contínua (CI/CD), disparando testes e deploys automaticamente. Exemplo prático: uma equipe pode centralizar todo o código de um sistema em um repositório CodeCommit e acompanhar sua evolução.

Git CI/CD

AWS Auto Scaling

Automatic Scaling

O AWS Auto Scaling ajusta automaticamente a quantidade de recursos computacionais conforme a demanda da aplicação. Ele cria novas instâncias em momentos de alta utilização e desliga quando o tráfego diminui, evitando desperdício de recursos. Esse mecanismo garante desempenho estável mesmo em picos inesperados de acesso. Exemplo prático: durante uma promoção em um site de vendas, o Auto Scaling adiciona instâncias EC2 extras para suportar o aumento de visitantes.
